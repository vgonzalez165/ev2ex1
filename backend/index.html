DOCUMENTACIÓN DE LA API PARA EL EXAMEN

POST /api/user

Registra un usuario en la aplicación.

Query String

Ninguno

Headers

Ninguno

Cuerpo del mensaje

JSON con los siguientes campos:

  • username: cadena con el nombre del usuario
  • pass: contraseña del usuario

Devuelve

Status 400: hay un error

Status 409: el nombre de usuario ya existe

Status 200 200 y JSON con los siguientes datos

  • id: identificador del usuario
  • token: webtoken para el usuario

GET /api/group

Query String

Es obligatorio el nombre de usuario y además admite dos filtros en la query string:

  • username: nombre de usuario
  • name: nombre o subcadena del nombre del grupo
  • category: categoría musical del grupo. Solo admite las categorías rap y rock

Headers

Se debe adjuntar el JWT en los headers

Cuerpo del mensaje

Ninguno

Devuelve

Status 200 y JSON con todos los grupos que se ajustan al filtro.
